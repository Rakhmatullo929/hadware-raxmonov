# Переключатель светлой/тёмной темы (Minimals-style)

**Дата:** 2026-07-13
**Статус:** утверждено, в работе

## Цель

Добавить полноценный переключатель темы «светлая ↔ тёмная», как в Minimals, с
корректно скорректированной тёмной палитрой. Тёмная тема — **по умолчанию** при
первом заходе. Переключатель — **в футере сайдбара** (рядом с языком) + компактная
плавающая кнопка на странице входа (там нет сайдбара).

## Механизм

Тема управляется атрибутом `data-bs-theme` на `<html>`:
- `data-bs-theme` включает нативный тёмный режим Bootstrap 5.3 (авто-инверсия
  `.btn-close`, дропдаунов и т.п.) **и** служит триггером для наших токенов.
- Тёмные значения объявляются блоком `:root[data-bs-theme="dark"] { … }`.
  Специфичность `0,2,0` выше и `:root` (`0,1,0`), и нативного `[data-bs-theme=dark]`
  Bootstrap (`0,1,0`) — иначе светлые значения из `:root` (грузится после Bootstrap)
  перебили бы тёмные.

Вся текущая тема построена на CSS-переменных (`--m-*`, `--bs-*`), поэтому тёмная тема —
это в основном переопределение токенов, без правок в разметке страниц.

## Тёмная палитра (значения Minimals)

- Поверхности: фон `#141A21`, бумага (карточки/сайдбар/топбар) `#1C252E`,
  приподнятая заливка `#28323D`.
- Текст: основной `#FFFFFF`, вторичный `#919EAB`, приглушённый `#637381`.
- Серые: `grey-100 #28323D`, `grey-200 #333d47`, `grey-300 #3a444e`,
  `grey-400 #4a555f`, `grey-500 #919EAB`, `grey-700 #DFE3E8`.
  `grey-900` **не** переопределяется (остаётся тёмным — тёмный текст на янтарной
  `.btn-warning`).
- Чип-заливки `*-lighter` → полупрозрачные тона `rgba(<цвет>, 0.16)`.
- Текст чипов/ссылок/outline-кнопок `*-dark` → светлые оттенки
  (primary `#5BE49B`, success `#77ED8B`, error `#FFAC82`, warning `#FFD666`,
  info `#61F3F3`, secondary `#C684FF`).
- Тени `--m-shadow-*` → на чёрной базе (`rgba(0,0,0,…)`) — серые тени на тёмном
  невидимы.

### Точечные исправления в тёмной теме
- `.app-topbar` фон `rgba(255,255,255,.85)` → `rgba(28,37,46,.8)`.
- `.lang-btn-active` фон `white` → тёмная поверхность.
- 4 hover-фона кнопок (`.btn-primary/success/danger/warning:hover`) фиксируются на
  исходных тёмных оттенках, т.к. токены `*-dark` теперь светлые.
- Утилиты: `.text-dark` → светлый текст, `.bg-light` → тёмная поверхность,
  `.table-light` → тёмные табличные переменные.
- Тинты строк посещаемости `.att-row-present/absent` → чуть плотнее (0.12).

### Правка разметки (1 файл)
- `customers/_table.html`: класс `bg-primary-bg-subtle` (не существует) → `bg-primary-subtle`,
  удалить инлайн `style="background:#eef2ff;color:#4338ca;"` — чип станет
  адаптивным через subtle-токены.

## Переключатель (UX)

- **Сайдбар:** сегментированный тумблер ☀️/🌙 (`[data-theme-value="light|dark"]`),
  стилистически как `.lang-switcher`. Активная кнопка подсвечивается.
- **Вход:** плавающая круглая кнопка `[data-theme-toggle]` вверху справа, иконка
  показывает тему, на которую переключит (в тёмной — солнце).
- Логика — `static/js/theme-toggle.js`: клик задаёт/переключает `data-bs-theme`,
  пишет `localStorage['rt-theme']`, синхронизирует активные состояния/иконки.
  Скрипт в `base.html`, поэтому переживает htmx-подмены партиалов.

## Сохранение и отсутствие мигания (FOUC)

- Выбор — в `localStorage['rt-theme']` (`light`/`dark`).
- Инлайн-скрипт в `<head>` (до отрисовки body) выставляет `data-bs-theme` из
  localStorage; пусто → `dark`. `<html data-bs-theme="dark">` — статический дефолт
  для случая без JS.

## Границы охвата

- Печать (`print_base.html`, `contract.html`, `return_receipt.html`) — всегда
  светлая (не наследует `data-bs-theme`, отдельный base). Профстандарт печати.
- Оверлеи модалок `rgba(0,0,0,.5)` — корректны на тёмном, не трогаем.

## Затрагиваемые файлы

- `static/css/theme.css` — блок тёмных токенов + точечные правки (основной объём).
- `templates/base.html` — head-скрипт, `<html data-bs-theme>`, тумблер в футере,
  плавающая кнопка для входа, подключение `theme-toggle.js`.
- `static/js/theme-toggle.js` — новый.
- `config/templates/config/customers/_table.html` — правка класса чипа.
