# Список «Аренды»: группировка по клиенту

**Дата:** 2026-07-14
**Статус:** утверждено, в работе

## Проблема

Список аренд — плоская таблица, одна строка = одна аренда, сортировка по сроку.
Клиент с несколькими арендами появляется несколько раз похожими строками
(имя/аватар одинаковые, отличается только `№`) → оператор путается.

## Решение

Группировать список по клиенту: клиент — заголовок группы с итогами, под ним —
его аренды. Клиент виден один раз.

### Бэкенд (`RentalListView`)
- Фильтры без изменений (статус, даты, поиск `q`, `customer`, `sort`).
- Пагинация: было 25 **аренд**/стр → 20 **клиентов**/стр (аренды клиента не
  разрываются между страницами).
- Порядок клиентов = по срочности их аренд. Реализация без GROUP BY (чтобы не
  конфликтовать с подзапросами-аннотациями): берём `values_list('customer_id')`
  из отфильтрованного и отсортированного набора аренд, дедупим с сохранением
  порядка — первый по сортировке (ближайший срок / просрочка) задаёт позицию
  клиента.
- Для клиентов текущей страницы подгружаем их аренды (`customer_id__in`),
  группируем в Python. Внутри группы порядок: просроченные → активные →
  закрытые, затем по сроку.
- Итоги группы (дёшево, из аннотаций): `count` аренд, `on_hand` = Σ
  `outstanding_total` (штук на руках), `overdue_count`. Денежный «долг» НЕ
  показываем — его нет дешёвой аннотацией.
- Пагинация — стандартный `Paginator` по списку `customer_ids`; в шаблон:
  `groups`, `page_obj`, `is_paginated` (партиал пагинации переиспользуется).

### Шаблон (`rentals/list.html`)
- Таблица с несколькими `<tbody class="rental-group">`: строка-заголовок
  `.group-head` (аватар, ФИО-ссылка, код, чипы «N аренд», «Σ шт на руках»,
  бейдж просрочки) + строки-аренды `.group-sub` (№, создана, срок, поз.,
  оплачено/залог, статус). Просроченные подстроки — красная полоса как сейчас.
- Сворачивание: клик по `.group-head` переключает `collapsed` на её `<tbody>`
  (CSS прячет `.group-sub`). По умолчанию развёрнуто. Немного JS в `extra_js`.
- Небольшая подпись сверху: «N клиентов · M аренд» для текущего фильтра.

### CSS (`theme.css`)
- Стили `.group-head` (фон-полоса, курсор, шеврон), `.group-sub` (отступ),
  `.rental-group.collapsed`. Токенами — корректно в обеих темах.

## Границы
- Денежный долг не считаем (дорого). Показываем штуки на руках + просрочку.
- Внутри группы показываем все статусы, что прошли фильтр статуса сверху.

## Проверка
- Рендер списка (по умолчанию, фильтр «просроченные», поиск клиента) в обеих
  темах; клиент с несколькими арендами — одна шапка + подстроки; пагинация по
  клиентам; сворачивание работает.
