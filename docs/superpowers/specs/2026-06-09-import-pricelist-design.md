# Импорт прайс-листа товаров (картинка → каталог)

**Дата:** 2026-06-09
**Ветка:** feat/autofill-due-date-clean (импорт делается отдельным коммитом)

## Цель

Перенести прайс-лист с двух картинок (тип товара · наименование/размер ·
цена за сутки) в каталог `core.Product`. Часть товаров уже заведена
data-миграцией `0016_seed_formwork_catalog` как «заготовки» с
`daily_price = 0` — им нужно проставить цену. Остальные позиции добавить.
Сделать без ручного ввода и без дублей/ошибок.

## Механизм

Новая data-миграция `config/migrations/0018_seed_pricelist.py`
(`RunPython`), по образцу `0016_seed_formwork_catalog`. При merge в `main`
прод деплоится автоматически — миграция применится сама.

«Цена за сутки» из картинки → поле `daily_price`. **Меняется только цена.**
`stock_total`, `deposit_per_unit` не трогаются.

### Сопоставление и идемпотентность

Товар ищется **по имени** (`get_or_create(name=...)`):

* найден → обновляю `daily_price` (и реактивирую, если был выключен);
* не найден → создаю: `unit='шт'`, `stock_total=0`, `deposit_per_unit=0`,
  `daily_price=<цена>`, `is_active=True`.

Нотация размера приводится к формату базы (как в `0016`): запятая → точка,
`x` → `×`. Напр. `2,2x40` → `Финская опалубка 2.2×40`. Финальные строки
имён зашиты в миграцию явно (без алгоритмической нормализации), чтобы
гарантированно совпасть с существующими именами.

`reverse` — `noop`: прежние цены восстановить нельзя, а слепое удаление
позиций опасно (FK `PROTECT` от истории аренд). Откат — вручную при
необходимости.

где цены различаются (4000/3000 против 1000).

## Деактивация (is_active=False, без удаления)

`Тахта`, `Стойка телескопическая домкрат`, `Леса строительные`, `Мешалка`
— обобщённые заготовки, заменяются конкретными размерами. Деактивация
безопасна: история аренд (`RentalItem.product`, FK `PROTECT`) сохраняется,
позиция просто исчезает из выбора при новых арендах (так же поступала
`0016`).
