# Дизайн: каталог опалубки + «Что входит в комплект»

Дата: 2026-06-07
Статус: утверждён (брейншторм)

## Цель

Завести в каталог детальные товары по размерам для опалубки и сопутствующих
групп, и для каждого товара указать **что входит в комплект** (зажим, фиксатор,
тайрод, штир/шайба, крючок, крестик) — **справочно**, для показа на странице
товара и в распечатке договора аренды.

Источник данных — фотографии прайс-листа, предоставленные пользователем
(корейская/финская опалубка, колонна, стойка телескопическая, леса строительные).

## Принятые решения (из брейншторма)

1. **Комплект — только справочно.** Не отдельные складские товары, не влияет на
   цену, залог, остаток. Просто текст.
2. **Каждая группа = отдельная категория.** Без новой модели подкатегории; схема
   иерархии не меняется (остаётся `Category → Product`).
3. **Цена/сутки, залог, остаток = 0** для новых товаров. Реальные значения
   пользователь проставит позже вручную.
4. **Старые обобщённые товары деактивируются** (`is_active=False`), история аренд
   сохраняется (на `Product` стоит `PROTECT`, удалять нельзя).
5. **Комплект показывается** на странице товара (форма/список) И в печати договора
   (`contract.html`).

## Архитектура

### Модель `Product` (config/models.py)

Добавить одно поле:

```python
included_kit = models.TextField("В комплекте", blank=True, default="")
```

- `blank=True` — необязательное (у большинства существующих товаров пусто).
- Хранит готовую строку, например:
  `Зажим ×3, Фиксатор ×3, Тайрод р/калпокча ×3, Штир/шайба ×3`.

*Почему текст, а не отдельная модель аксессуаров:* комплект используется только
для показа/печати, поэтому структурированная таблица `KitComponent` была бы
избыточной (YAGNI). Текст редактируется как обычный textarea.

### Форма `ProductForm` (config/forms.py)

Добавить `included_kit` в `Meta.fields`. Шаблон формы
(`products/form.html`) перебирает все поля формы автоматически — отдельной правки
шаблона формы не требуется (поле отрендерится как textarea с Bootstrap-классом,
как остальные поля). Проверить, что в `__init__` формы Bootstrap-класс
навешивается и на Textarea-виджеты.

### Отображение

- **Список товаров** (`products/_table.html`): под названием товара (или в
  отдельной ячейке) показать «В комплекте: …», если поле непустое.
- **Печать договора** (`config/templates/config/rentals/contract.html`): под
  `{{ it.product.name }}` в обеих таблицах (накладная без цен и с ценами) добавить
  мелкой строкой курсивом `в комплекте: {{ it.product.included_kit }}`, если поле
  непустое.

### Наполнение — миграция данных

Новая миграция данных в `config/migrations/` (паттерн как у `0002_seed_catalog`,
`0012_seed_norm_products`). Идемпотентная:

- Категории создаются через `Category.objects.get_or_create(name=...)`.
- Товары — через `Product.objects.get_or_create(name=..., defaults={...})` с
  `category`, `unit='шт'`, `daily_price=0`, `deposit_per_unit=0`, `stock_total=0`,
  `included_kit=...`.
- Деактивация старых: `Product.objects.filter(name__in=[...]).update(is_active=False)`.
- `reverse_code` — мягкий откат (по желанию `noop`), чтобы миграция была обратимой
  без падения.

Миграция использует `apps.get_model('config', 'Product'/'Category')` (историческая
модель), не импортирует модель напрямую.

## Данные

Все размеры в названии — через `×` (как у существующего «Щит опалубки 1.2×0.6»).
Названия товаров с префиксом группы.

## Тестирование

- Тест миграции/сидирования: после `migrate` в БД есть 5 категорий и 47 товаров с
  ожидаемым `included_kit`; старые 4 товара `is_active=False`.
- Тест модели: поле `included_kit` существует, по умолчанию пустая строка.
- Тест формы: `included_kit` присутствует в форме и сохраняется.
- Тест печати: `contract.html` отрисовывает «в комплекте: …» для товара с
  непустым `included_kit` и ничего не показывает для пустого.
- Существующие тесты (`tests/`) не падают.

## Вне рамок (YAGNI)

- Отдельная модель/таблица аксессуаров комплекта.
- Складской учёт аксессуаров, влияние на цену/залог.
- Модель подкатегории (двухуровневая иерархия).
- Реальные цены/залоги/остатки (заполняются вручную позже).
- Автоматическое списание аксессуаров при выдаче.
