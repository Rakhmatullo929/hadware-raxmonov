# Модалка «Создать аренду» на странице клиента

**Дата:** 2026-07-12
**Статус:** одобрено, в реализации

## Цель
На детальной странице клиента — кнопка «Создать аренду». По клику открывается
модалка с **полной** формой создания аренды, где клиент уже выбран и не
меняется. После успешного создания — переход на страницу новой аренды.

## Подход
Переиспользовать существующую форму создания (не дублировать):
- Тело `<form>` из `create.html` выносим в общий партиал `_rental_form.html`
  (шапка, позиции, итоги). Инклюдится и полной страницей, и модалкой.
- Расчётный JS (дни × сумма, авто-срок) из инлайна `create.html` выносим в
  `static/js/rental-create.js` — идемпотентный, на делегировании событий +
  `htmx:afterSettle`, работает и на странице, и в модалке.

## Компоненты
- `_rental_form.html` — общий внутренний контент формы.
- `_create_rental_modal.html` — оболочка модалки поверх `_rental_form.html`;
  клиент показан только для чтения + скрытый `name="customer"`.
- `static/js/rental-create.js` — вынесенный расчётный JS.
- `RentalCreateView` — модальный режим:
  - `GET ?customer=<pk>` (htmx) → рендер модалки с этим клиентом.
  - `POST` (htmx) успех → заголовок `HX-Redirect` на `rental_detail`;
    ошибка → перерисовать модалку с ошибками.
- Страница клиента: кнопка «Создать аренду» → `hx-get` в новый слот
  `#create-modal-slot` (отдельный от `#modal-slot` карточек аккордеона);
  подключение `rental-create.js`.

## Крайние случаи
- Архивный клиент → кнопка скрыта (сначала разархивировать).
- Ошибка валидации → модалка перерисовывается на месте, аренда не создаётся.
- Права: staff/admin (как у создания аренды).

## Тесты
- GET модалки: клиент зафиксирован, скрытый `customer`=pk, есть слот позиций.
- POST валидный → аренда создана для этого клиента + `HX-Redirect` на аренду.
- POST невалидный → модалка с ошибкой, аренда не создана.
- Кнопка скрыта для архивного клиента.
- Полный набор тестов остаётся зелёным (поведение полной страницы не меняется).
