# Водяной знак-логотип на заднем фоне при печати договора

**Дата:** 2026-06-02
**Статус:** реализовано (с отступлением от исходного дизайна — см. примечание)

> **Примечание (обновление после реализации):** в коде поехал не круг с буквой
> «R», описанный ниже, а диагональная **текстовая** надпись «Raxmonov»
> (`static/img/logo.svg` — это `<text>`; в PDF — строка
> `_WATERMARK_TEXT='Raxmonov'` в `config/contract_pdf.py`, коммит 22bfedd).
> Расположение/диагональ/бледность остались как в спеке; сменилась только форма
> знака. Текст ниже описывает первоначальный замысел (круг + «R») и оставлен
> для истории.

## 1. Цель

При печати договора аренды показывать на заднем фоне бледный чёрно-белый
логотип («водяной знак»), не мешающий чтению. Знак должен выглядеть одинаково
на обоих путях печати:

* **HTML-печать из браузера** — страница `contract.html`, печать через
  `window.print()`.
* **Серверный PDF** — `GET /rentals/<pk>/contract.pdf`, сборка через `fpdf2`.

Параметры внешнего вида (согласовано):

* **Расположение:** один крупный знак по центру листа, **повёрнут по диагонали**.
* **Цвет:** серый / чёрно-белый, **малозаметный** (~8–10% «плотности»).
* **Режим:** всегда включён при печати, без переключателя.
* **Документы:** только договор аренды (единственный печатный документ сейчас).

## 2. Выбранный подход (Вариант A)

Нативная отрисовка в каждом из путей с общим простым знаком. Без новых
зависимостей (в духе проекта: «чистый Python, без системных библиотек»).
Знак намеренно простой — окружность + буква «R», — чтобы его можно было
одинаково воспроизвести и в SVG (HTML), и примитивами `fpdf2` (PDF).

Отвергнутые альтернативы:

* **Единый PNG для обоих путей** — требует Pillow/бинарник в репозитории,
  «запечённую» прозрачность тяжело подстраивать, в PDF поворот/масштаб
  картинки и альфа капризнее.
* **Только текст** — пользователь просил логотип, а не просто надпись.

## 3. Компоненты

### 3.1 Ассет логотипа — `static/img/logo.svg`

Монохромный знак: окружность + буква «R» по центру. Один цвет
(`currentColor` / чёрный), без фоновой заливки страницы. Бледность задаётся
не самим файлом, а параметрами отрисовки (CSS-`opacity` в HTML, светло-серый
цвет в PDF), чтобы знак легко донастраивать. Файл переиспользуем и для
будущего брендинга.

### 3.2 HTML-печать — `templates/print_base.html`

Это общий базовый шаблон всех печатных страниц, поэтому правка здесь
автоматически распространится на будущие печатные документы.

* Добавить **элемент `<img>`** (не CSS `background-image`!) перед блоком
  `.print-page`:
  `<img class="print-watermark" src="{% static 'img/logo.svg' %}" alt="" aria-hidden="true">`.
  Причина выбора `<img>`: браузеры по умолчанию **не печатают** CSS-фон, если
  пользователь вручную не включил «Фоновую графику»; передний `<img>`
  печатается всегда.
* Стили `.print-watermark`:
  `position: fixed; top: 50%; left: 50%;`
  `transform: translate(-50%, -50%) rotate(-30deg);`
  `width: 60%; max-width: 16cm; opacity: .08; filter: grayscale(1);`
  `z-index: 0; pointer-events: none;`
* `.print-page { position: relative; z-index: 1; }` — контент поверх знака.
* Знак виден и на экране-превью (чтобы пользователь видел результат), и при
  печати (`@media print` ничего не скрывает у `.print-watermark`).
* Размер уменьшается на малых форматах: для `print-page--half` и
  `print-page--quarter` задать меньшую `width` (например, 45%).
* Подключить `{% load static %}` в `print_base.html`, если ещё не подключён.

### 3.3 Серверный PDF — `config/contract_pdf.py`

В класс `_ContractPDF` (внутри `_make_contract_pdf`) добавить метод
`header()`. `fpdf2` вызывает `header()` в начале **каждой** страницы, **до**
отрисовки контента, — поэтому знак гарантированно ложится **под** текст.

Метод `header()`:

1. Сохраняет текущие позицию, цвет текста/линий и шрифт.
2. В контексте `with self.rotation(angle=..., x=cx, y=cy):` (где `cx, cy` —
   центр листа, считается от `self.w` и `self.h`) рисует знак светло-серым.
   **Направление наклона выбрать так, чтобы визуально совпадало с HTML**
   (в HTML — `rotate(-30deg)`; в `fpdf2` положительный угол вращает в другую
   сторону, поэтому знак угла подбирается при реализации, ориентир ±30°):
   * окружность — `self.set_draw_color(205, 205, 205)` + `self.circle(...)`
     (или `self.ellipse(...)`), толщина линии задаётся `self.set_line_width`;
   * букву «R» — `self.set_text_color(205, 205, 205)`, шрифт `Body` `B`
     крупного размера, отцентрованную в окружности.
   Радиус знака — доля от `min(self.w, self.h)` (например, ~0.35), чтобы
   корректно работать на A4/A5/A6.
3. Восстанавливает позицию курсора на верхнее поле (`set_xy(l_margin, t_margin)`)
   и прежние цвет/шрифт, чтобы вёрстка контента не сдвинулась.

Светло-серый цвет даёт «малозаметность» без возни с альфа-каналом для
изображений в `fpdf2`. Знак появляется на всех страницах многостраничного
договора (например, на `full` с переносами таблицы).

## 4. Поток данных

Новых данных и моделей нет. Водяной знак — чисто презентационный слой:

* HTML: статический `<img>` + CSS, без участия view.
* PDF: рисуется в `header()` на каждой странице во время сборки документа;
  никаких изменений в сигнатуре `build_contract_pdf(rental, size)`.

## 5. Обработка ошибок и крайние случаи

* **Отсутствует `logo.svg`** (HTML): сломанная картинка не должна портить
  печать — `alt=""` + бледность делают её незаметной; ассет коммитим в репо,
  так что в норме он есть.
* **Многостраничный PDF:** знак на каждой странице (механизм `header()`).
* **Малые форматы (A5/A6):** размер знака масштабируется от размеров листа
  (HTML — через классы формата, PDF — через долю от `min(w, h)`).
* **Сдвиг вёрстки PDF:** обязателен возврат курсора в `header()`; покрывается
  тестом «PDF валиден и непуст» для всех трёх форматов.
* **Шрифт PDF:** «R» рисуется уже загруженным шрифтом `Body`, отдельных
  зависимостей не добавляет.

## 6. Тестирование

* **PDF (`tests/`):** `build_contract_pdf()` для `full` / `half` / `quarter`
  по-прежнему возвращает непустой валидный PDF (начинается с `%PDF-`).
  Подтверждает, что `header()`/поворот не ломают сборку и вёрстку.
* **HTML (`tests/`):** ответ `rental_contract` (`GET /rentals/<pk>/contract/`)
  содержит элемент с классом `print-watermark` и ссылку на `img/logo.svg`.

## 7. Вне объёма (YAGNI)

* Переключатель «печатать со знаком / без» — решено: всегда включён.
* Модель компании и загрузка собственного логотипа.
* Водяной знак на других документах/отчётах — сейчас печатается только договор
  (HTML-сторона получит знак «бесплатно» через `print_base.html`).

## 8. Затрагиваемые файлы

* `static/img/logo.svg` — **новый** ассет.
* `templates/print_base.html` — элемент `.print-watermark` + стили.
* `config/contract_pdf.py` — метод `header()` в `_ContractPDF`.
* `tests/` — проверки PDF (3 формата) и HTML (наличие знака).
